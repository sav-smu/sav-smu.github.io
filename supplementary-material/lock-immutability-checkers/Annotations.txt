Note: java.util.concurrent package contains “implementations designed for use in multithreaded contexts: ConcurrentHashMap, ConcurrentSkipListMap, ConcurrentSkipListSet, CopyOnWriteArrayList, and CopyOnWriteArraySet” (according to the Java SE 8 API specifications). Therefore we manually extend the annotation 
@!entailLocking(java.util.concurrent) that is automatically extracted to five separate annotations, i.e., @!entailLocking(ConcurrentHashMap), @!entailLocking(ConcurrentSkipListMap), @!entailLocking(ConcurrentSkipListSet), @!entailLocking(CopyOnWriteArrayList), @!entailLocking(CopyOnWriteArraySet), according to the specification sentence “A concurrent collection is thread-safe, but not governed by a single exclusion lock.”.
